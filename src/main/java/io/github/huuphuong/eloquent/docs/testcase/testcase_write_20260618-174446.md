## Behavioral Notes

- `create(...)` inserts all simple properties and skips a null primary key value.
- `insertBatch(...)` groups inserts by column signature and executes them in batch.
- `insertBatchIgnoreError(...)` returns `false` when any batch insertion fails.
- `update(...)` updates all simple properties except the primary key.
- `updatePartial(...)` and map-based partial updates only touch requested properties.
- `upsert(...)` creates when the primary key is missing and falls back to create when update affects zero rows.
- Pivot mutation methods were verified with mocked `NamedParameterJdbcTemplate` interactions.
- SQL preview methods were verified to generate the expected insert, delete, and sync statements without database writes.
